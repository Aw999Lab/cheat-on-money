# /money-status — 状态看板 + 自动路由

## 核心铁律

1. **看完状态直接行动。** 展示完不等人说下一步——AI 自己判断该做什么，直接路由。
2. **不列选项清单。** 不展示"你可以说 X、Y、Z"。选最优动作，直接执行。

## 流程

### Step 1 — 读状态
```bash
test -f .money-state.json && echo OK || echo MISSING
```
MISSING → 自动路由 money-init。

### Step 2 — 展示看板（精简）
- 画像 + 段位
- 已沉淀经验摘要（lessons.md）
- 候选机会列表（按可行→存疑→高危排序）
- 当前执行中机会 + 进度
- 复盘记录摘要

### Step 3 — 自动路由（核心——不看用户眼色）

| 状态 | 自动动作 |
|------|---------|
| 无候选机会 | → 立即路由 money-find |
| 有可行未选 | → 自动选最高分 → 路由 money-plan |
| 有执行中的 ≥ 14 天 | → 自动路由 money-retro |
| 有执行中的 < 14 天 | → 展示进度，给本周目标，继续 |
| 所有机会都测完了 | → 路由 money-find 找新方向 |

**展示 + 路由一气呵成。** 用户说"看板" → 展示状态 → 自动判断下一步 → 直接执行。
