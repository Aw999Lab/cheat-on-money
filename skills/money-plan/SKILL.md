# /money-plan — 出方案 + 直接执行

## ⛔ 最高铁律

**方案写完后禁止停止。必须立即执行验证第一步。**
- 不准问"你想开始吗"
- 不准等用户确认方案
- 能做的直接做掉：注册、发帖、写代码、部署、挂服务
- 不能做的（如需要手机扫码）：准备好所有材料，说"只差这一步，材料已备好"

## 前置
- 若未指定机会 → 读 `.money-state.json` 取 `active.chosen_id`
- 高危 → 跳过，选次优
- 存疑 → 以验证为第一目标

## 方案 5 块（精简输出）

1. **验证第一步**：最小动作 + AI 立即代劳
2. **前两周动作**：每天做什么，可量化
3. **收入预期**：多久见钱、多少
4. **止损线**：何时停
5. **AI 提效点**：哪些环节 AI 做

## 落盘 + 自动执行

1. 写方案到 `.money-state.json` 的 `active`（plan + prediction）
2. **立即执行验证第一步**——打开浏览器、写代码、生成文案、注册账号
3. 只汇报进展："我已经帮你做了 X，结果 Y。接下来 Z。"

## 执行完成后

继续监控。到了验证节点（≥14天 或 用户说"复盘"）→ 自动触发 money-retro。
